# 버스/오토바이 클릭 → 노선도·정류장 오버레이 상세기획

> 기준일: 2026-06-24 · Figma: node 1933-50387 · 트리거: **클릭**(확정) · 대상: **버스·오토바이 동일**(확정)

---

## 1. 개요 / 인터랙션 플로우

지도 위 버스·오토바이 마커 **클릭** 시:
1. **배경 딤** (기존 구현 재사용)
2. 클릭 마커 + 핀 **정확한 모양으로 딤 제외** (기존 cutout 재사용)
3. **DRT 운행정보 모달** 표출 (기존 구현, 위치만 노선 시작점에 맞춤)
4. **노선도 오버레이 신규**: 형광 녹색 점선 노선 + 번호 정류장 핀(1~6) + 노선 위 차량 글로우
5. 딤/바깥 클릭 → 전체 닫힘 (기존)

> 모달과 노선도는 **하나의 노선 데이터**를 공유 → 모달의 "정류장 6개 / 차량 2대"와 오버레이의 정류장·차량 수가 항상 일치.

---

## 2. 데이터 모델 (단일 소스)

```
ROUTE = {
  id: 'gm-circular',
  name: '광명역세권 순환',
  status: '운행중',
  vehicles: 2,
  // 지도(map-stage) % 좌표 기준 경로 점들 (도로 위에 맞춤)
  path: [ {x, y}, {x, y}, ... ],     // 형광 점선이 지나는 경로
  stations: [
    { no: 1, name: '1번 정류장', x, y },
    ... 6개
  ]
}
```
- 좌표는 **map-stage 비율(%)** → 지도 확대/축소·반응형에서 자동 정렬.
- 추후 실제 DRT 노선 좌표로 교체만 하면 됨.

---

## 3. 화면 구성 (레이어 z-index)

| 레이어 | z-index | 내용 |
|---|---|---|
| 딤 backdrop | 8998 | 반투명 검정 (기존) |
| **노선 오버레이 SVG** | 8999 | 형광 점선 경로 + 정류장 핀 + 라벨 (신규) |
| 차량 cutout (딤 제외) | 8999 | 클릭 차량+핀 원본 모양 (기존, 글로우 추가) |
| DRT 모달 | 9000 | 운행정보 카드 (기존) |

- 노선 오버레이 SVG는 **map-stage와 동일 좌표계**(viewBox 1920×3699)로 깔아 정렬.
- 정류장 핀: 흰 원+번호(Figma), 우측에 "N번 정류장" 라벨 칩(backdrop-blur 흰 배경).

---

## 4. 비주얼 (Figma 반영)

- **점선 노선**: stroke `#06A85B`, dash, `filter: drop-shadow` 형광 글로우(0 0 5px, 0 0 20px #06a85b).
- **정류장 핀**: 원형 흰 배경 + 녹색 테두리 + 번호, 하단 글로우 점, 우측 라벨 칩.
- **차량 글로우**: 클릭 차량에 녹색 글로우(box-shadow 0 0 5/20px #06a85b) — Figma의 버스 글로우.
- **모달**: 기존 DRT 카드 (노선/정류장/차량/상세보기).

---

## 5. 애니메이션

- **점선 흐름**: `stroke-dashoffset` 애니메이션으로 점선이 진행 방향으로 흐름 → "운행중" 생동감.
- 정류장 핀 **순차 페이드인**(stagger) → 노선이 그려지는 느낌(선택).
- 차량 글로우 **은은한 펄스**(선택).

---

## 6. 인터랙션

- 노선/정류장은 기본 표시 전용. **상세보기** → 모빌리티 데이터 지도 이동(기존).
- (확장) 정류장 클릭 → 도착 예정/대기 정보 팝오버 — Phase 2 옵션.

---

## 7. 더 나은 방향 제안

1. **현재 위치·다음 정류장 강조** — 차량의 현재 위치와 "다음 정류장"을 하이라이트(ETA 표시)하면 단순 노선도보다 정보 가치↑. (추천)
2. **점선 방향 흐름 + 차량 진행** — 점선이 흐르고 차량 글로우가 노선 위를 따라가면 실시간 운행감. (추천)
3. **노선 경로 정확도** — (a) 데모 경로(좌표 데이터, 도로 위 수동 정렬, 빠름·조정 용이) vs (b) Lottie 버스 모션패스 추출(실제 주행 경로와 정확히 일치, 작업량 큼). **추천: (a)로 시작 → UX 확정 후 필요 시 (b)로 업그레이드.**
4. **버스 vs 오토바이** — 확정대로 동일 노선도 사용. 단, 라벨을 "노선/정류장"으로 통일하되 오토바이는 향후 "배달 거점"으로 라벨만 분기 가능(옵션).
5. **정류장 클릭 도착정보**(Phase 2) — 시민 활용도↑.

## 8. 작업 단계 & 검증

| 단계 | 내용 | 검증 |
|---|---|---|
| 0 | 노선 데이터(경로/정류장 좌표) 정의 — 지도 도로 위 정렬 | 헤드리스 렌더로 경로가 도로 위에 앉는지 |
| 1 | 노선 오버레이 SVG(점선+정류장+라벨) 렌더 | 딤 위 정렬·비주얼 |
| 2 | 점선 흐름 애니메이션 + 차량 글로우 | 동작 |
| 3 | 클릭 플로우 통합(딤·모달·cutout·노선도 동시) | 클릭→전체 표출, 닫기 |
| 4 | (옵션) 현재위치/다음정류장, 정류장 클릭 정보 | — |

각 단계 헤드리스 스크린샷 검증.
